# ESG EXPORT 수출역량진단 시스템

수출역량 진단지를 **컨설턴트가 상황별로 수정**해서, **링크를 제3자(고객)에게 전송**하면
고객이 직접 진단을 작성하고 그 결과가 **Google Sheet 고객DB에 자동 누적**되는 시스템입니다.

- `index.html` — 진단 페이지 (컨설턴트 편집 + 고객 응답 겸용)
- `google-apps-script.gs` — Google Sheet에 결과를 저장하는 백엔드 코드

---

## 전체 흐름

```
①  컨설턴트가 index.html 을 열어 문항·인프라 항목을 상황에 맞게 수정
        │
②  [🔗 진단 링크 생성] 클릭 → 수정한 진단지가 그대로 담긴 링크 생성·복사
        │
③  카카오톡·이메일 등으로 고객(제3자)에게 링크 전송
        │
④  고객이 링크 접속 → 기업정보·연락처 입력 후 진단 응답
        │
⑤  진단 완료 시 결과가 Google Sheet "고객DB" 시트에 자동 저장 (고객DB 누적)
```

---

## 1. Google Sheet + 백엔드 설치 (최초 1회)

1. [Google 스프레드시트](https://sheets.new)에서 새 시트를 하나 만듭니다. (예: `수출역량진단 고객DB`)
2. 상단 메뉴 **[확장 프로그램] → [Apps Script]** 를 엽니다.
3. 기본 `Code.gs` 내용을 모두 지우고, **`google-apps-script.gs` 파일 내용을 그대로 붙여넣기** 한 뒤 저장합니다.
4. 우측 상단 **[배포] → [새 배포]** 클릭
   - 유형: **웹 앱**
   - 실행 계정: **나(본인)**
   - 액세스 권한: **모든 사용자(Anyone)**
   - **[배포]** → 권한 승인
5. 표시되는 **웹 앱 URL** 을 복사합니다. (`https://script.google.com/macros/s/.../exec`)

## 2. index.html 에 백엔드 URL 연결

`index.html` 안에서 아래 부분을 찾아 복사한 웹 앱 URL로 교체합니다.

```js
const GOOGLE_SCRIPT_URL = 'https://script.google.com/macros/s/.../exec';
```

> 이미 운영 중인 URL이 들어 있다면 그대로 두어도 됩니다.

## 3. 페이지 호스팅

`index.html` 을 인터넷에서 접속 가능한 주소에 올립니다. (GitHub Pages, 회사 홈페이지 등)
파일 하나만 있으면 동작합니다.

## 컨설턴트 사용법

1. 진단 페이지를 엽니다.
2. **⚙ 문항 관리** 에서 7개 영역·문항 내용을 상황에 맞게 수정 → **저장**.
   - **수출 인프라 현황 체크리스트** 항목도 자유롭게 추가/삭제/수정할 수 있습니다.
3. **🔗 진단 링크 생성** 클릭 → **[📋 링크 복사]** → 고객에게 전송.
   - 생성된 링크에는 **수정한 문항·체크리스트가 그대로 포함**됩니다.
   - 수정 후에는 **반드시 다시 링크를 생성**해야 변경 내용이 반영됩니다.

## 고객(제3자) 화면

- 링크(`?mode=client`)로 접속하면 **고객 자가진단 모드**로 열립니다.
  - 관리 버튼·컨설턴트 의견 편집 화면은 보이지 않습니다.
  - **기업명·응답자 성함·이메일/연락처**는 결과 안내를 위해 필수 입력입니다.
- 진단을 마치면 결과(점수·등급·레이더차트)가 표시되고,
  **자동으로 Google Sheet 고객DB에 저장**됩니다.

---

## Google Sheet 에 저장되는 항목

`고객DB` 시트에 행 단위로 누적됩니다.

| 구분 | 컬럼 |
|------|------|
| 접수정보 | 접수시각, 진단일자, 구분(고객 자가진단/컨설턴트 진단) |
| 고객DB | 기업명, 담당자, 직책, **이메일**, **연락처** |
| 기업개요 | 업종, 주요제품, 매출규모, 수출경험, 해외지사, 부서별 인원 |
| 진단결과 | 종합점수, 등급, 7개 영역별 점수, 인프라 요약 |
| 상세 | 컨설턴트의견, 전체데이터(JSON) |

> 문항을 바꿔도 `전체데이터(JSON)` 컬럼에 모든 응답이 보존되어 데이터 유실이 없습니다.

## 참고 / 주의

- 전송은 `no-cors` 방식이라 브라우저 콘솔에 결과 응답이 보이지 않지만 저장은 정상 동작합니다.
  설치 후 **테스트 응답을 한 번 제출**해 시트에 행이 쌓이는지 확인하세요.
- 링크에 문항 데이터가 포함되므로 문항 수가 매우 많아지면 링크가 길어질 수 있습니다(정상).
- 개인정보(이메일·연락처)를 수집하므로, 고객 안내 시 활용 목적(맞춤 컨설팅)을 함께 고지하는 것을 권장합니다.
